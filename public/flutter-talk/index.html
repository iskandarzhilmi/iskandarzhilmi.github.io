Tool 02

Sentry

Error tracking for real-world apps

Second: error tracking. Pain: angry users, losing revenue, no idea how or why, can't debug

Sentry

A must once you have real users

  • Real error logs from real devices
  • Catch crashes you'd never see
  • Lifesaver on Android camera quirks
Pricing: Free tier 5K errors/month. Paid from $26/month.
Logs straight from users' phones, crashes after updates ship, every Android camera has quirks. Used even by million-user apps. Andrea's example: missing url_launcher intent broke every Android link, Sentry caught it day one. Free tier gotcha: layout errors fire every frame, 5K burns fast

Sentry dashboard

Device, OS, app version, stack trace

Sentry Feedback

In-app bug report form. Pairs with the feedback package: screenshot + annotate, lands in Sentry

Tool 04

Shorebird

Push Flutter code updates over the air

OTA patches, skip review

Shorebird

Why Shorebird?

  • Push updates without a new build
  • Bypasses app store submission review
  • Great for small UX improvements, no store update
Small UX patches, no rebuild, no review queue

Shorebird

Limitations

  • Patch downloads on launch, applies on the next restart
  • Dart only: assets, native code & engine changes can't be patched
  • Not suitable for critical bug fixes
Two opens to take effect, Dart only

Tool 05

force_update_helper

Prompt users to update when a new version ships

by Andrea Bizzotto (the Flutter legend!)

Fifth: forcing updates, by Andrea

force_update_helper

Why force updates?

  • Fatal errors & DTO mismatches with the backend happen
  • Old versions in the wild keep breaking silently
  • Host the minimum version on a GitHub Gist or backend
  • I prefer Gist: less backend-dependent, easier solo
Fatal errors, DTO mismatches. Gist: backend could be down, URL changes, wrong URL in Flutter. Solo = simpler. On a team: agree on who owns the gist, avoid future issues

Example

force_update_config.json

{
  "config": {
    "minimum_version": "1.4.6"
  }
}

Hosted on a GitHub Gist

One JSON file = kill switch

Ratings done right
